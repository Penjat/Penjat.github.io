A* Pathfinding

This is my implementation of the A* pathfinding algorithm, a very common algorithm for all types of games. What I find extremely fascinating about this algorithm, is that the same algorithm can work on any type of grid. In my example, the exact same code can find a path on a square, hexagonal, and triangular grids. The same algorithm could easily be used to find paths on 3D, 4D+, or nodular grids. The only information it requires is: 1)How many steps it took to get to a tile and 2)How long it ideally would take to get to the end.
C# - Unity3D - A* Algorithm

Android Metronome

This turned into a larger project than I initially planned. I was able to make a simple metronome quite quickly, but I found that of speeds over ~210 bpm the metronome was no longer accurate. The play function on the media player was not immediate enough to be accurate at those speeds. I had previously noticed as a music teacher that many online JavaScript metronomes seemed to be slightly off. I ended up solving the problem by parsing the binary data from the sample sound and creating a new wav file every time the tempo was changed, and simply looping through that. This results in an extremely accurate metronome, my version has an arbitrary limit at 600bpm, but I found it to be working fine at 1000s of beat per minute.
Java - Native Android

Monty Hall Simulation

This was a quick build and like many great works, was inspired by ignorance on the internet. I was amazed that in 2018 I was still encountering people who were arguing about the logic behind the Monty Hall problem. I thought that, rather than explain it with math, why don't I just let the user try it as a simulation and see the results for themselves. The web app lets the user play the game the regular way, or lets the user run a large number of simulations. The app then provides an analysis of the data (which door was better to bet on) based on all the simulations run. With a small sample size, the results are quite random, but with enough simulations, the data converges on the expected probabilities.
JavaScript - HTML - CSS
